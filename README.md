# Agile Runner

동료들의 성장을 돕는 AI 코드 리뷰 코치

## 프로젝트 소개 🌟

Agile Runner는 GitHub의 Pull Request 이벤트를 감지하여 자동으로 **AI 코드 리뷰**를 제공하는 GitHub App 기반 서비스입니다. Spring Boot와 OpenAI의 GPT 4o-mini 모델을 활용해 PR의 코드 변경 내용을 분석하고, 리뷰어 인원이 부족한 상황에서도 **즉각적이고 부담 없는 피드백**을 제공합니다. 

특히 리뷰어를 직접 확보해야 하는 환경에서, 빠르게 개선점을 발견하고 성장할 수 있도록 돕는 것을 목표로 합니다.

## 주요 기능 🧩

- **자동 PR 리뷰 생성:** GitHub Webhook을 통해 PR 오픈/재오픈 이벤트를 감지하고 AI 기반의 코드 리뷰를 자동 생성합니다.
- **AI 코드 변경 분석:** Spring AI (OpenAI API 연동)을 통해 Pull Request의 diff(변경 코드)를 분석하고, 리뷰 총평과 최대 5개의 인라인 코멘트를 작성합니다.
- **리뷰 결과 GitHub 등록:** 생성된 총평 및 인라인 코멘트를 GitHub API를 통해 해당 PR의 리뷰 코멘트로 등록합니다 (JSON 응답을 파싱하여 GitHub에 전달).
- **정교한 Diff 파싱:** GitHub의 Patch/Hunk/Line/Position을 자체 구현한 diff 파서로 처리하여, **정확한 위치에 인라인 코멘트**를 달아줍니다. 변경된 각 파일의 패치 내용과 코멘트 가능 위치를 계산해, AI가 제공한 라인 번호를 정확한 GitHub 코드 위치로 변환합니다.
- **컨테이너 및 클라우드 배포:** Dockerfile 기반으로 Render에 컨테이너를 배포할 수 있도록 구성되어 있습니다.

## 기술 스택 ⚙️

- **Backend:** Java 21, Spring Boot (Spring Web), Lombok
- **AI:** Spring AI OpenAI Starter를 이용한 ChatGPT API 연동
- **GitHub API:** GitHub App + GitHub REST API (org.kohsuke GitHub Java SDK 활용), JGit
- **Auth:** JWT (JSON Web Token)를 이용한 GitHub App 인증 (앱 ID 및 Private Key를 통해 Installation Token 발급)
- **Deployment:** Docker (Gradle 기반 빌드 및 JAR 패키징), Render Cloud (무료 플랜으로 자동 배포 설정)
- **Testing:** JUnit 5, AssertJ를 활용한 도메인 로직 테스트

## 프롬프트 설계 철학 📝

Agile Runner의 AI 프롬프트는 **교육적인 코드 리뷰어**의 페르소나로 설계되었습니다. 

“함께 고민하는 동료”라는 톤앤매너를 유지하며, 리뷰 내용과 형식을 일관되게 생성합니다. 

주요 원칙은 다음과 같습니다:

- **정답 제공 금지:** 리뷰에서는 문제 코드에 대한 **직접적인 정답이나 최종 코드**를 제시하지 않습니다. 전체 코드를 다시 작성해주거나 단순 정정하는 대신, "A 방향으로 개선해보면 어떨까요?"와 같이 **힌트와 제안** 위주의 피드백을 제공합니다. 이는 참가자가 **스스로 생각하며 해결책을 찾도록 돕기 위한 설계**입니다.
- **'우리' 시점의 어조:** 일방적인 지적을 피하고 **포용적인 화법**을 사용합니다. 리뷰어와 개발자가 한 팀이라는 느낌을 주기 위해 "*당신의 코드*" 대신 "*우리 코드*", "*함께 개선해보아요*"와 같은 표현을 씁니다. 이러한 **협력적 어조**를 통해 피드백에 대한 심리적 부담을 줄이는 것을 목표로 합니다.
- **따뜻한 제안형 코멘트:** 어떠한 경우에도 **인신공격이나 단정적인 표현을 하지 않으며**, 항상 긍정적인 표현과 질문형 제안을 우선합니다. 예를 들어 "*이 부분이 잘못됐네요.*" 대신 "*이 부분을 다른 방식으로 바꿔보면 어떨까요?*"처럼 개선 방향을 함께 고민하는 어투로 작성합니다.
- **JSON 형식 응답:** AI 모델은 리뷰 결과를 **JSON 형식**으로 반환하도록 프롬프트 되어있습니다. JSON 객체에는 **`body`** 필드에 전체 리뷰 총평을 담고, **`inlineComments`** 필드에 최대 5개의 인라인 코멘트를 배열로 제공합니다. 각 인라인 코멘트는 해당 파일의 `path`, 코드 `line` 번호, 그리고 코멘트 내용인 `body`가 포함됩니다. 이 구조 덕분에 응답을 GitHub API 호출에 바로 매핑하기 쉽고, 일관된 리뷰 형식을 유지할 수 있습니다. *(Agile Runner는 리뷰 생성 시 항상 이 구조를 따르도록 설계되었습니다.)*
- **인라인 코멘트 최대 5개:** 하나의 PR에 너무 많은 지적 사항을 남기지 않기 위해, **인라인 코멘트는 최대 5개로 제한**합니다. **가장 핵심적인 개선점 몇 가지에 집중**하기 위함이며, AI 리뷰어가 과도한 피드백으로 사용자에게 부담을 주지 않도록 합니다. 필요 이상의 코멘트는 내부 로직에서 제한하거나 필터링합니다.
- **구조화된 리뷰 내용:** 프롬프트는 리뷰를 **인사 및 공감 → 잘한 점 칭찬 → 개선 사항 제안**의 흐름으로 작성하도록 유도합니다. 먼저 짧은 격려 인사로 리뷰를 시작하고, PR에서 발견한 장점 2~4가지를 구체적으로 칭찬합니다. 이후 개선이 필요한 핵심 사항 2~3가지를 선정해 맥락과 중요성을 설명하고, 개선 방향을 질문 형태로 제안합니다. 이러한 구조는 **참가자의 노력과 성장을 모두 존중하는 균형 잡힌 리뷰**를 생성하기 위함입니다.

## 향후 개선 방향 🔮

Agile Runner는 현재 기본 기능에 초점을 맞춰 구현되어 있으며, 

앞으로 다음과 같은 개선을 계획하고 있습니다:

- **다양한 언어 및 미션 지원:** 현재는 주로 Java 기반의 프리코스 미션 맥락에 최적화되어 있지만, 추후 Python, JavaScript 등 **여러 언어와 과제 유형에 대한 코드 리뷰 규칙**을 추가하여 범용성을 높일 예정입니다. 상황에 따라 프롬프트를 유연하게 변경하거나 사용자 설정을 통해 다른 코드 스타일 가이드에도 대응할 수 있도록 개선할 계획입니다.
- **AI 모델 및 프롬프트 튜닝:** 리뷰 품질 향상을 위해 OpenAI 모델의 **세부 매개변수 조정**이나 시스템 프롬프트 개선을 지속적으로 진행할 예정입니다. 필요하다면 학습된 지식을 최신 상태로 유지하거나, 조직별 코드 스타일을 반영한 **커스텀 프롬프트**를 도입하는 것도 검토하고 있습니다.
- **성능 및 비용 최적화:** 대규모 리포지토리의 PR이나 변경 파일이 많은 경우에도 원활히 동작할 수 있도록, diff를 파일/청크 단위로 나누어 처리하는 **스트리밍 방식**과 변경 요지를 단계적으로 압축하는 **요약 전략** 도입을 검토하고 있습니다. 또한 OpenAI API 호출 빈도를 줄이고, 동일한 내용에 대한 반복 리뷰를 피하기 위한 캐싱을 통해 응답 속도와 비용 효율을 함께 개선할 계획입니다.
- **추가 통합 및 기능:** GitHub PR 코멘트 등록 이외에도 Slack이나 이메일로 주요 피드백을 요약 전달하는 등 **알림 연동**을 검토하고 있습니다. 더 나아가 GitHub Actions와 연계해 PR 생성 시 자동으로 Agile Runner를 호출하는 Workflow 템플릿을 제공하거나, VS Code와 같은 IDE에서 바로 Agile Runner를 활용할 수 있는 방안도 고려 중입니다.
- **코드 실행 기반 피드백:** 향후에는 테스트 코드나 샌드박스 환경에서 **PR의 코드를 직접 실행해 보는 기능**도 검토하고 있습니다. 예를 들어 제출된 코드의 테스트를 자동으로 실행하고 그 결과를 리뷰에 반영하면, **런타임 에러나 실패하는 테스트 케이스에 대한 보다 구체적인 피드백**을 제공할 수 있습니다. (향후 실험을 위해 Testcontainers 등 관련 의존성을 일부 추가해 둔 상태입니다.)

Agile Runner를 통해 개발자들이 **신속하고 친절한 코드 리뷰**를 받으며 지속적으로 성장할 수 있기를 바랍니다. 💪 앞으로도 다양한 개선을 통해 더 나은 코드 리뷰 경험을 제공하도록 발전해 나가겠습니다!
