# 현재 활성 단계

## 문서 목적
이 문서는 현재 활성 구현 범위와 후속 구현 범위를 고정하는 현재 단계 문서다.
검증 기준과 작업 문서는 이 문서의 활성 범위를 기준으로 작성하며, GitHub 이슈는 각 작업을 외부에서 추적하기 위한 수단으로만 사용한다.

## 현재 활성 단계
### ID
SPEC-0030

### 이름
운영용 조회 응답 준비 데이터 반복 검증 스크립트 초안 구현

### 목표
- 운영용 조회 응답 대표 검증 절차에 실제 초안 파일을 추가한다.
- 현재 보조 명령 가이드 기준 절차에서 정리한 입력/출력, 종료 흐름, 인계 지점을 초안 파일에 실제로 반영한다.
- 반복 검증 명령 실행은 초안 파일로 옮기고, 응답 의미 해석과 H2 결과 판단은 계속 수동 단계로 남긴다.

### 대상 문제
- SPEC-0029에서 파일 후보 구조, 입력/출력 계약, 종료 흐름, 수동 확인 경계는 정리했지만, 실제 초안 파일은 아직 없다.
- 문서만으로 반복 검증을 따라갈 수는 있지만, seed 적용, rerun/retry 요청, 실행 키 추출, H2 조회를 여전히 사람이 명령 단위로 반복해야 한다.
- 이 상태가 계속되면 대표 검증 반복 비용과 실수 가능성이 그대로 남는다.

### 범위
- 운영용 조회 응답 대표 검증 절차를 실제 스크립트 초안 파일로 옮긴다.
- 아래 내용을 최소 범위로 포함한다.
  - `prepare-seed.sh`, `run-rerun.sh`, `run-retry.sh`, `collect-evidence.sh` 초안 파일 추가
  - 입력 인자와 출력 파일 실제 반영
  - 파일별 종료 흐름과 수동 확인 인계 지점 실제 반영
  - 대표 검증에 쓸 수 있는 최소 실행 흐름 확보
  - 구현 결과와 다음 단계 위험 정리
- 현재 운영용 조회 응답 세트에 적용한다.
  - `rerun`
  - `retry`
  - 단건 조회
  - 목록 조회
  - 이력 조회
  - 관리자 조치 응답

### 비대상
- 새 경로 추가
- 응답 필드 추가 또는 이름 변경
- 준비 데이터 SQL 자체 구조 변경
- 운영자 UI

### 외부 계약
- 기존 rerun/retry/단건 조회/목록 조회/이력 조회/관리자 조치 경로와 HTTP 상태 계약은 유지한다.
- 기존 응답 필드 의미는 바꾸지 않는다.
- 이번 단계는 대표 검증 스크립트 초안 파일을 실제로 추가하는 작업이며, 관리 API 계약 자체를 바꾸는 작업이 아니다.

### 핵심 시나리오
1. 구현 시작 전 기존 기준과 안전망 확인
   - `SPEC-0029` 단계 요약 문서와 현재 가이드 문서가 실제 초안 파일 구현 시작 근거로 충분한지 먼저 확인한다.
   - 부족한 경우만 최소 문서 보강을 검토한다.
2. 준비와 정리 초안 파일 구현
   - `prepare-seed.sh`에 시작 전 점검, 정리 SQL 실행, 준비 데이터 적용 SQL 실행 흐름을 옮긴다.
   - 입력 인자와 출력 파일을 현재 문서 기준과 맞춘다.
3. rerun, retry, 실행 근거 수집 초안 파일 구현
   - `run-rerun.sh`, `run-retry.sh`, `collect-evidence.sh`에 대표 검증 명령 흐름을 옮긴다.
   - 종료 흐름과 출력 파일 생성, 수동 확인 인계 지점을 실제 파일에 반영한다.
4. 초안 파일 대표 검증과 단계 마감
   - 실제 초안 파일로 대표 재실행/재시도 흐름을 실행한다.
   - 출력 파일, 응답, H2 실행 근거, 문서 기준이 서로 맞는지 확인하고 마감 근거를 남긴다.

### 작업 분해 기준
- `TASK-0001` 초안 구현 시작 전 기존 기준과 안전망 확인
- `TASK-0002` 준비와 정리 초안 파일 구현
- `TASK-0003` rerun, retry, 실행 근거 수집 초안 파일 구현
- `TASK-0004` 초안 파일 대표 검증과 단계 마감

### 연결 검증 기준
- `manual-rerun-script-draft-safety-net-preserved`
- `manual-rerun-script-draft-prepare-implemented`
- `manual-rerun-script-draft-run-flow-implemented`
- `manual-rerun-script-draft-representative-verified`

### 필수 테스트 시나리오
- 초안 파일을 추가하는 동안 기존 rerun/retry/단건 조회/목록 조회/이력 조회/관리자 조치 자동 검증은 유지된다.
- `prepare-seed.sh`, `run-rerun.sh`, `run-retry.sh`, `collect-evidence.sh`가 현재 문서 기준 입력/출력과 같은 흐름을 가진다.
- 대표 재실행/재시도 검증에서 초안 파일이 남긴 출력 파일과 H2 실행 근거를 같은 실행 키 기준으로 다시 읽을 수 있어야 한다.

## 후속 단계 후보
### ID
SPEC-0031

### 이름
운영용 조회 응답 준비 데이터 반복 검증 스크립트 적용 검증 정리

### 시작 조건
- `현재 활성 단계`가 완료되고, 초안 파일 대표 검증 결과가 마감된 뒤 시작한다.

### 목표
- `현재 활성 단계`에서 추가한 초안 파일을 대표 검증 결과와 함께 더 쉽게 적용하고 유지할 수 있게 정리한다.

### 후속 변경 범위
- 초안 파일 적용 순서 정리
- 대표 검증 결과와 출력 파일 매핑 보강
- 초안 파일 사용 가이드 정리

### 후속 변경 비대상
- 사용자용 UI
- 장기 저장소 도입

### 후속 검증 방향
- 현재 구현 결과를 바탕으로 초안 파일 적용 절차와 유지 기준을 더 정리한다.
