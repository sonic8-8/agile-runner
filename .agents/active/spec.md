# 현재 활성 단계

## 문서 목적
이 문서는 현재 활성 구현 범위와 후속 구현 범위를 고정하는 현재 단계 문서다.
검증 기준과 작업 문서는 이 문서의 활성 범위를 기준으로 작성하며, GitHub 이슈는 각 작업을 외부에서 추적하기 위한 수단으로만 사용한다.

## 현재 활성 단계
### ID
SPEC-0036

### 이름
운영용 조회 응답 반복 검증 스크립트 실패 사례 빠른 참조 적용 예시 정리

### 목표
- 이미 정리한 빠른 참조 카드를 실제 실패 상황에 붙여, 운영자가 어떤 카드를 먼저 펼쳐야 하는지 더 빨리 따라가게 만든다.
- 빠른 참조 카드에서 상세 실패 사례 예시 표와 보조 문서로 내려가는 실제 사용 예시를 정리한다.
- 빠른 참조 문서가 실제 사용 순서를 돕되, 상세 가이드 역할 자체를 대신하지 않도록 경계를 유지한다.

### 대상 문제
- `SPEC-0035`로 빠른 참조 카드 자체는 생겼지만, 실제 실패 상황에서 어느 카드를 먼저 펼쳐야 하는지 바로 대입하는 예시는 부족하다.
- 운영자는 카드 종류를 알게 됐어도, 특정 실패 상황에서 어디로 먼저 진입하고 다음에 어느 상세 예시 표나 보조 문서로 내려가야 하는지 다시 추론할 수 있다.
- 빠른 참조와 상세 문서가 모두 준비됐지만, 실제 사용 순서를 붙여 주는 적용 예시가 없어 첫 대응 속도가 아직 느릴 수 있다.

### 범위
- 운영용 조회 응답 반복 검증 스크립트 실패 사례 빠른 참조 적용 예시를 정리한다.
- 아래 내용을 최소 범위로 포함한다.
  - 적용 예시 정리 전에 빠른 참조 문서와 테스트 다시 확인
  - 종료 코드 빠른 참조 적용 예시 정리
  - 출력 파일 누락과 H2 조회 실패 적용 예시 정리
  - 적용 예시 마감과 실제 사용 순서 연결 정리
- 현재 운영용 조회 응답 반복 검증 세트에 적용한다.
  - `manual-rerun-response-seed-command-guide.md`
  - `prepare-seed.sh`
  - `run-rerun.sh`
  - `run-retry.sh`
  - `collect-evidence.sh`
  - 관련 회고와 자동 검증 테스트

### 비대상
- 새 스크립트 구현
- 실제 앱/H2 대표 검증 재실행
- 기존 종료 코드, 출력 파일 이름, 응답 계약 변경
- 운영자용 API 계약 변경
- 관리자 UI
- 빠른 참조 카드 자체의 새 유형 추가

### 외부 계약
- 기존 재실행, 재시도, 단건 조회, 목록 조회, 이력 조회, 관리자 조치 경로와 HTTP 상태 계약은 유지한다.
- 기존 스크립트 이름, 종료 코드, 출력 파일 이름은 이번 단계에서 바꾸지 않는다.
- 이번 단계는 빠른 참조 적용 예시 정리 작업이지, 대표 검증 절차나 상세 실패 사례 문서, 빠른 참조 카드 자체의 기준을 다시 쓰는 단계가 아니다.

### 핵심 시나리오
1. 적용 예시 정리 전에 빠른 참조 문서와 테스트 다시 확인
   - `SPEC-0035` 단계 요약과 현재 빠른 참조 문서, 자동 검증 테스트를 다시 읽고 돌려 이번 단계 시작 전에 같은 기준을 다시 잡는다.
   - 부족한 경우만 최소 문서 보강을 검토한다.
2. 종료 코드 실패 빠른 참조 적용 예시 정리
   - 종료 코드 실패를 만났을 때 어느 빠른 참조 표를 먼저 보고, 다음에 어느 상세 예시 표로 내려가는지 짧은 적용 예시로 정리한다.
3. 출력 파일 누락과 H2 조회 실패 적용 예시 정리
   - 출력 파일 누락 카드와 H2 조회 실패 카드를 실제 실패 상황에 어떻게 먼저 적용하는지 예시로 정리한다.
4. 적용 예시 마감과 실제 사용 순서 연결 정리
   - 적용 예시를 본 뒤 실제 사용 순서를 문서로 다시 따라갈 수 있게 마지막 연결과 확인 질문을 남긴다.

### 작업 분해 기준
- `TASK-0001` 적용 예시 정리 전에 빠른 참조 문서와 테스트 기준 확인
- `TASK-0002` 종료 코드 빠른 참조 적용 예시 정리
- `TASK-0003` 출력 파일 누락과 H2 조회 실패 적용 예시 정리
- `TASK-0004` 적용 예시 마감과 실제 사용 순서 연결 정리

### 연결 검증 기준
- `manual-rerun-script-application-example-safety-net-preserved`
- `manual-rerun-script-stop-code-application-example-documented`
- `manual-rerun-script-output-missing-and-h2-application-example-documented`
- `manual-rerun-script-application-example-closeout-documented`

### 필수 테스트 시나리오
- 적용 예시 정리 동안 기존 재실행, 재시도, 단건 조회, 목록 조회, 이력 조회, 관리자 조치 자동 검증은 유지된다.
- 운영자가 실제 실패 상황을 떠올렸을 때 어느 빠른 참조 표 또는 카드를 먼저 펼쳐야 하는지 문서만 보고 다시 고를 수 있어야 한다.
- 빠른 참조와 상세 문서가 서로 다른 역할을 가지며, 문서만 보고 다음에 어느 상세 예시 표와 보조 문서로 내려가야 하는지 다시 찾을 수 있어야 한다.

## 후속 단계 후보
### ID
SPEC-0037

### 이름
운영용 조회 응답 반복 검증 스크립트 실패 사례 빠른 참조 적용 유지 기준 정리

### 시작 조건
- `현재 활성 단계`가 완료되고, 빠른 참조 적용 예시 문서가 정리된 뒤 시작한다.

### 목표
- 빠른 참조 적용 예시를 정리한 뒤, 이후 문서와 예시가 어긋날 때 어디를 먼저 고쳐야 하는지 유지 기준을 더 직접적으로 정리한다.

### 후속 변경 범위
- 빠른 참조 적용 예시 유지 기준
- 적용 예시와 상세 문서 어긋남 점검 순서
- 빠른 참조 적용 예시 마감 체크리스트

### 후속 변경 비대상
- 사용자용 UI
- 장기 저장소 도입

### 후속 검증 방향
- 현재 구현 결과를 바탕으로 운영자가 빠른 참조 적용 예시를 유지할 때 어디를 먼저 고쳐야 하는지 더 빨리 판단할 수 있게 한다.
