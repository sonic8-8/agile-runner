# 현재 활성 단계

## 문서 목적
이 문서는 현재 활성 구현 범위와 후속 구현 범위를 고정하는 현재 단계 문서다.
검증 기준과 작업 문서는 이 문서의 활성 범위를 기준으로 작성하며, GitHub 이슈는 각 작업을 외부에서 추적하기 위한 수단으로만 사용한다.

## 현재 활성 단계
### ID
SPEC-0028

### 이름
운영용 조회 응답 준비 데이터 반복 검증 스크립트 초안 검토

### 목표
- 운영용 조회 응답 대표 검증 절차에서 실제 스크립트 초안으로 묶을 최소 범위를 검토한다.
- 현재 보조 명령 가이드 기준 절차에서 어떤 명령 묶음을 초안 범위로 삼고, 어떤 단계는 계속 수동 확인으로 남길지 정리한다.
- 스크립트 초안의 입력/출력, 중단 조건, 비대상을 문서로 고정해 다음 단계 구현 여부를 판단할 근거를 남긴다.

### 대상 문제
- SPEC-0027에서 스크립트 도입 가치와 경계는 정리했지만, 실제 초안을 만든다면 어디까지를 한 번에 실행 대상으로 묶을지와 어디서 멈춰야 하는지는 아직 문서에 없다.
- 준비 데이터 정리/적용, 앱 기동, 대표 요청, 실행 키 추출, 앱 종료, H2 조회를 모두 한 덩어리로 만들면 실패 지점이 흐려질 수 있다.
- 반대로 초안 범위를 너무 좁게 잡으면 반복 비용을 줄이는 효과가 약해져 스크립트 검토 의미가 떨어진다.

### 범위
- 운영용 조회 응답 대표 검증 절차를 실제 스크립트 초안으로 묶을 최소 범위를 검토한다.
- 아래 내용을 최소 범위로 포함한다.
  - 실제 초안으로 묶을 명령 묶음 후보 정리
  - 초안 입력 값, 출력 파일, 중간 산출물 경계 정리
  - 초안이 멈춰야 하는 실패 조건과 계속 수동으로 남길 확인 단계 정리
  - 문서 기반 절차와 초안 기반 절차가 함께 있을 때의 전환 기준 정리
  - 다음 단계 구현으로 넘어갈지 판단하는 마감 근거
- 현재 운영용 조회 응답 세트에 적용한다.
  - `rerun`
  - `retry`
  - 단건 조회
  - 목록 조회
  - 이력 조회
  - 관리자 조치 응답

### 비대상
- 실제 스크립트 구현
- 새 경로 추가
- 응답 필드 추가 또는 이름 변경
- 준비 데이터 SQL 자체 구조 변경
- 운영자 UI

### 외부 계약
- 기존 rerun/retry/단건 조회/목록 조회/이력 조회/관리자 조치 경로와 HTTP 상태 계약은 유지한다.
- 기존 응답 필드 의미는 바꾸지 않는다.
- 이번 단계는 대표 검증 스크립트 초안 범위를 검토하는 작업이며, 관리 API 계약 자체를 바꾸는 작업이 아니다.

### 핵심 시나리오
1. 초안 검토 시작 전 기존 판단과 근거 확인
   - `SPEC-0027` 단계 요약 문서와 현재 가이드 문서가 스크립트 초안 검토 단계의 시작 근거로 충분한지 먼저 확인한다.
   - 부족한 경우만 최소 문서 보강을 검토한다.
2. 초안 범위와 입력/출력 파일 정리
   - 준비 데이터 정리/적용, 앱 기동, 대표 요청, 실행 키 추출, 앱 종료, H2 조회 중 어디까지를 초안 한 번에 묶을지 정리한다.
   - 입력 값, 출력 파일, 중간 산출물과 계속 수동으로 남길 결과 읽기 단계를 함께 정리한다.
3. 실패 중단 조건과 수동 확인 유지 조건 정리
   - 어떤 실패에서 초안이 바로 멈춰야 하는지, 어떤 확인은 사람이 끝까지 직접 닫아야 하는지 정리한다.
   - H2 잠금, 응답 값 해석, H2 의미 해석, 회고 작성은 어떻게 남길지 함께 정리한다.
4. 초안 검토 판단과 다음 단계 마감
   - 현재는 초안 검토로 충분한지, 아니면 후속 단계에서 실제 스크립트 구현 검토로 넘어가야 하는지 판단 근거를 남긴다.
   - 대상 테스트와 전체 테스트를 다시 돌려 문서 정리로 기존 검증 흐름이 흔들리지 않았는지 확인한다.

### 작업 분해 기준
- `TASK-0001` 초안 검토 시작 전 기존 판단과 근거 확인
- `TASK-0002` 초안 범위와 입력/출력 파일 정리
- `TASK-0003` 실패 중단 조건과 수동 확인 유지 조건 정리
- `TASK-0004` 초안 검토 판단과 단계 마감

### 연결 검증 기준
- `manual-rerun-script-draft-safety-net-preserved`
- `manual-rerun-script-draft-boundary-documented`
- `manual-rerun-script-draft-stop-condition-documented`
- `manual-rerun-script-draft-closeout-completed`

### 필수 테스트 시나리오
- 스크립트 초안 검토 문서를 정리하는 동안 기존 rerun/retry/단건 조회/목록 조회/이력 조회/관리자 조치 자동 검증은 유지된다.
- 현재 보조 명령 가이드와 회고만으로 대표 검증 절차와 초안 후보 경계를 다시 재구성할 수 있어야 한다.
- 어떤 명령 묶음을 초안 범위로 보고 어떤 실패 조건과 확인 단계는 수동으로 남기는지 문서에서 구분되어야 한다.

## 후속 단계 후보
### ID
SPEC-0029

### 이름
운영용 조회 응답 준비 데이터 반복 검증 스크립트 초안 구현 검토

### 시작 조건
- `현재 활성 단계`가 완료되고, 대표 검증 보조 명령이 문서화된 뒤 시작한다.

### 목표
- `현재 활성 단계`에서 초안 범위가 충분히 좁혀졌다고 판단되면, 실제 스크립트 초안 구현 검토로 넘어간다.

### 후속 변경 범위
- 보조 스크립트 초안 구현 범위 검토
- 입력/출력 파일과 실행 경계 확정
- 문서 기반 절차와 초안 기반 절차의 실제 전환 기준 정리

### 후속 변경 비대상
- 사용자용 UI
- 장기 저장소 도입

### 후속 검증 방향
- 현재 검토 결과를 바탕으로 실제 초안이 필요한지 판단하고, 필요하면 그 초안의 최소 범위를 정한다.
