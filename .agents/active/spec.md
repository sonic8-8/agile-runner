# 현재 활성 단계

## 문서 목적
이 문서는 현재 활성 구현 범위와 후속 구현 범위를 고정하는 현재 단계 문서다.
검증 기준과 작업 문서는 이 문서의 활성 범위를 기준으로 작성하며, GitHub 이슈는 각 작업을 외부에서 추적하기 위한 수단으로만 사용한다.

## 현재 활성 단계
### ID
SPEC-0034

### 이름
운영용 조회 응답 반복 검증 스크립트 실패 사례 예시 정리

### 목표
- 반복 검증 스크립트에서 실제로 만날 수 있는 실패 사례를 예시로 정리해, 운영자가 문서만 보고도 어떤 실패인지 빠르게 비교할 수 있게 한다.
- `SPEC-0033`에서 잠근 오류 대응 기준 위에 실제 실패 문구와 예시 값을 붙여, 해석 기준과 예시가 한 흐름으로 이어지게 한다.
- 실패 사례 예시가 정상 흐름 가이드와 유지 기준 문서의 역할을 침범하지 않도록 경계를 분명히 한다.

### 대상 문제
- `SPEC-0033`까지 오면서 종료 코드 해석, 출력 파일 누락 점검, H2 잠금과 코드 오류 분리 기준은 정리됐지만, 처음 보는 운영자는 실제 실패 문구와 예시 값을 바로 대조할 자료가 없다.
- 지금 상태에서는 스크립트 로그 문구나 출력 파일 누락 상황을 문서 기준과 직접 연결하려면 기존 회고와 로그를 다시 뒤져야 한다.
- 오류 대응 기준이 있어도 실제 예시가 없으면, 운영자는 같은 종료 코드라도 어떤 실패 사례에 해당하는지 다시 추론해야 한다.

### 범위
- 운영용 조회 응답 반복 검증 스크립트 실패 사례 예시를 정리한다.
- 아래 내용을 최소 범위로 포함한다.
  - 실패 사례 예시 정리 시작 전 기존 근거와 안전망 확인
  - 종료 코드와 멈춤 실패 사례 예시 정리
  - 출력 파일 누락 실패 사례 예시 정리
  - H2 잠금 실패 사례 예시와 단계 마감
- 현재 운영용 조회 응답 반복 검증 세트에 적용한다.
  - `prepare-seed.sh`
  - `run-rerun.sh`
  - `run-retry.sh`
  - `collect-evidence.sh`
  - `manual-rerun-response-seed-command-guide.md`
  - 관련 회고와 자동 검증 테스트

### 비대상
- 새 스크립트 구현
- 실제 앱/H2 대표 검증 재실행
- 기존 종료 코드나 출력 파일 이름 변경
- 운영자용 API 계약 변경
- 관리자 UI

### 외부 계약
- 기존 재실행, 재시도, 단건 조회, 목록 조회, 이력 조회, 관리자 조치 경로와 HTTP 상태 계약은 유지한다.
- 기존 스크립트 이름, 종료 코드, 출력 파일 이름은 이번 단계에서 바꾸지 않는다.
- 이번 단계는 실패 사례 예시 정리 작업이지, 대표 검증 절차를 새로 늘리거나 운영 API 계약을 고치는 단계가 아니다.

### 핵심 시나리오
1. 실패 사례 예시 정리 시작 전 기존 근거와 안전망 확인
   - `SPEC-0033` 마감 근거와 현재 오류 대응 기준 문서, 자동 검증이 실패 사례 예시 정리를 시작하기에 충분한지 먼저 확인한다.
   - 부족한 경우만 최소 문서 보강을 검토한다.
2. 종료 코드와 멈춤 실패 사례 예시 정리
   - 종료 코드별로 실제로 어떤 실패 문구와 상황을 예시로 읽을지 정리한다.
3. 출력 파일 누락 실패 사례 예시 정리
   - 필요한 출력 파일이 없을 때 어떤 누락 사례로 해석하는지 예시를 정리한다.
4. H2 잠금 실패 사례 예시와 단계 마감
   - H2 잠금 의심 사례를 실제 로그 문구와 질문 흐름으로 정리하고 단계 마감 근거를 남긴다.

### 작업 분해 기준
- `TASK-0001` 실패 사례 예시 정리 시작 전 기존 근거와 안전망 확인
- `TASK-0002` 종료 코드와 멈춤 실패 사례 예시 정리
- `TASK-0003` 출력 파일 누락 실패 사례 예시 정리
- `TASK-0004` H2 잠금 실패 사례 예시와 단계 마감

### 연결 검증 기준
- `manual-rerun-script-failure-example-safety-net-preserved`
- `manual-rerun-script-stop-code-examples-documented`
- `manual-rerun-script-output-missing-examples-documented`
- `manual-rerun-script-h2-lock-example-closeout-documented`

### 필수 테스트 시나리오
- 실패 사례 예시 정리 동안 기존 재실행, 재시도, 단건 조회, 목록 조회, 이력 조회, 관리자 조치 자동 검증은 유지된다.
- 운영자가 종료 코드와 멈춤 실패 사례를 문서만으로 다시 비교할 수 있어야 한다.
- 운영자가 출력 파일 누락 사례와 H2 잠금 사례를 문서만으로 다시 비교할 수 있어야 한다.

## 후속 단계 후보
### ID
SPEC-0035

### 이름
운영용 조회 응답 반복 검증 스크립트 실패 사례 빠른 참조 정리

### 시작 조건
- `현재 활성 단계`가 완료되고, 실패 사례 예시가 정리된 뒤 시작한다.

### 목표
- 실패 사례 예시를 더 짧은 참조 카드나 빠른 비교 표로 정리해, 운영자가 긴 문서를 전부 읽지 않고도 먼저 확인 대상을 좁힐 수 있게 한다.

### 후속 변경 범위
- 실패 사례 예시 빠른 참조 표
- 종료 코드, 출력 파일 누락, H2 잠금 비교 축약본
- 오류 대응 문서와 빠른 참조 문서 연결

### 후속 변경 비대상
- 사용자용 UI
- 장기 저장소 도입

### 후속 검증 방향
- 현재 구현 결과를 바탕으로 운영자가 실패 유형별 비교를 더 빠르게 할 수 있게 한다.
