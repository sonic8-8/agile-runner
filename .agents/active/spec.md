# 현재 활성 단계

## 문서 목적
이 문서는 현재 활성 구현 범위와 후속 구현 범위를 고정하는 현재 단계 문서다.
검증 기준과 작업 문서는 이 문서의 활성 범위를 기준으로 작성하며, GitHub 이슈는 각 작업을 외부에서 추적하기 위한 수단으로만 사용한다.

## 현재 활성 단계
### ID
SPEC-0035

### 이름
운영용 조회 응답 반복 검증 스크립트 실패 사례 빠른 참조 정리

### 목표
- 긴 실패 사례 문서를 다 읽기 전에 운영자가 종료 코드, 출력 파일 누락, H2 조회 실패 유형을 먼저 좁힐 수 있는 빠른 참조 카드를 만든다.
- `SPEC-0034`의 상세 실패 사례 예시를 축약해, 대표 검증 중 멈춤 지점을 더 빨리 분류하게 만든다.
- 빠른 참조 문서가 상세 가이드 역할을 침범하지 않도록, 먼저 보는 카드와 다음에 보는 상세 문서의 구분을 분명히 한다.

### 대상 문제
- `SPEC-0034`로 실제 실패 문구와 예시 값은 충분히 정리됐지만, 문서가 길어서 운영자가 처음 실패를 만났을 때 바로 비교 대상을 좁히기 어렵다.
- 지금은 종료 코드, 출력 파일 누락, H2 조회 실패를 빠르게 가르는 짧은 카드가 없어, 운영자는 먼저 긴 가이드를 열고 필요한 구간을 다시 찾아야 한다.
- 반복 검증 절차는 문서화됐지만, 현장에서 먼저 보는 축약 표가 없어 실패 분류 속도가 느리다.

### 범위
- 운영용 조회 응답 반복 검증 스크립트 실패 사례 빠른 참조를 정리한다.
- 아래 내용을 최소 범위로 포함한다.
  - 빠른 참조 정리 시작 전 기존 근거와 안전망 확인
  - 종료 코드 실패 사례 빠른 참조 정리
  - 출력 파일 누락 빠른 참조 카드 정리
  - H2 조회 실패 빠른 참조 카드 정리
  - 빠른 참조 마감과 상세 문서 연결 정리
- 현재 운영용 조회 응답 반복 검증 세트에 적용한다.
  - `manual-rerun-response-seed-command-guide.md`
  - `prepare-seed.sh`
  - `run-rerun.sh`
  - `run-retry.sh`
  - `collect-evidence.sh`
  - 관련 회고와 자동 검증 테스트

### 비대상
- 새 스크립트 구현
- 실제 앱/H2 대표 검증 재실행
- 기존 종료 코드, 출력 파일 이름, 응답 계약 변경
- 운영자용 API 계약 변경
- 관리자 UI

### 외부 계약
- 기존 재실행, 재시도, 단건 조회, 목록 조회, 이력 조회, 관리자 조치 경로와 HTTP 상태 계약은 유지한다.
- 기존 스크립트 이름, 종료 코드, 출력 파일 이름은 이번 단계에서 바꾸지 않는다.
- 이번 단계는 실패 사례 빠른 참조 정리 작업이지, 대표 검증 절차나 상세 실패 사례 문서의 기준 자체를 다시 쓰는 단계가 아니다.

### 핵심 시나리오
1. 빠른 참조 정리 시작 전 기존 근거와 안전망 확인
   - `SPEC-0034` 마감 근거와 현재 실패 사례 예시 문서, 자동 검증이 빠른 참조 정리를 시작하기에 충분한지 먼저 확인한다.
   - 부족한 경우만 최소 문서 보강을 검토한다.
2. 종료 코드 실패 사례 빠른 참조 정리
   - 종료 코드 10부터 33까지 어떤 실패 유형을 먼저 좁혀야 하는지 짧은 비교 표로 정리한다.
3. 출력 파일 누락과 H2 조회 실패 빠른 참조 정리
   - 출력 파일 누락 카드와 H2 조회 실패 카드를 따로 만들고, 각각 어떤 질문으로 빠르게 구분하는지 정리한다.
4. 빠른 참조 마감과 상세 문서 연결 정리
   - 빠른 참조를 본 뒤 상세 문서 어느 위치로 내려가야 하는지 연결하고, 마지막 확인 질문과 단계 마감 근거를 남긴다.

### 작업 분해 기준
- `TASK-0001` 빠른 참조 정리 전에 이전 실패 예시 문서와 테스트 기준 확인
- `TASK-0002` 종료 코드 실패 사례 빠른 참조 정리
- `TASK-0003` 출력 파일 누락과 H2 조회 실패 빠른 참조 정리
- `TASK-0004` 빠른 참조 마감과 상세 문서 연결 정리

### 연결 검증 기준
- `manual-rerun-script-quick-reference-safety-net-preserved`
- `manual-rerun-script-stop-code-quick-reference-documented`
- `manual-rerun-script-output-missing-and-h2-quick-reference-documented`
- `manual-rerun-script-quick-reference-closeout-documented`

### 필수 테스트 시나리오
- 빠른 참조 정리 동안 기존 재실행, 재시도, 단건 조회, 목록 조회, 이력 조회, 관리자 조치 자동 검증은 유지된다.
- 운영자가 긴 실패 사례 문서를 전부 읽지 않고도 종료 코드, 출력 파일 누락, H2 조회 실패를 먼저 좁힐 수 있어야 한다.
- 빠른 참조와 상세 문서가 서로 다른 역할을 가지며, 문서만 보고 어디서 상세 문서로 내려가야 하는지 다시 찾을 수 있어야 한다.

## 후속 단계 후보
### ID
SPEC-0036

### 이름
운영용 조회 응답 반복 검증 스크립트 실패 사례 빠른 참조 적용 예시 정리

### 시작 조건
- `현재 활성 단계`가 완료되고, 빠른 참조 문서가 정리된 뒤 시작한다.

### 목표
- 빠른 참조 문서를 실제 실패 예시와 함께 적용하는 예시를 추가해, 운영자가 어떤 상황에서 어느 참조 카드를 먼저 펼쳐야 하는지 더 빨리 따라가게 한다.

### 후속 변경 범위
- 빠른 참조 적용 예시
- 대표 실패 유형별 첫 진입 예시
- 빠른 참조와 상세 문서 이동 예시

### 후속 변경 비대상
- 사용자용 UI
- 장기 저장소 도입

### 후속 검증 방향
- 현재 구현 결과를 바탕으로 운영자가 실패 유형별 첫 분류를 더 빠르게 수행할 수 있게 한다.
