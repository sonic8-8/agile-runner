# 현재 활성 단계

## 문서 목적
이 문서는 현재 활성 구현 범위와 후속 구현 범위를 고정하는 현재 단계 문서다.
검증 기준과 작업 문서는 이 문서의 활성 범위를 기준으로 작성하며, GitHub 이슈는 각 작업을 외부에서 추적하기 위한 수단으로만 사용한다.

## 현재 활성 단계
### ID
SPEC-0037

### 이름
운영용 조회 응답 반복 검증 스크립트 실패 사례 빠른 참조 적용 유지 기준 정리

### 목표
- 이미 정리한 빠른 참조 적용 예시를 나중에 고칠 때, 무엇을 먼저 보고 무엇을 같이 고쳐야 하는지 더 직접적으로 정리한다.
- 적용 예시, 상세 예시 표, 보조 문서가 어긋날 때 어디서부터 다시 확인해야 하는지 순서로 남긴다.
- 실제 스크립트나 응답 계약 변경 없이도 운영자가 문서 수정 순서를 더 빨리 판단하게 만든다.

### 대상 문제
- `SPEC-0036`으로 실제 실패 상황에 바로 대입할 적용 예시는 생겼지만, 이후 문서와 예시가 어긋날 때 어디를 먼저 고쳐야 하는지 기준은 아직 부족하다.
- 운영자는 적용 예시를 고치려 할 때 빠른 참조, 상세 예시 표, 보조 명령 가이드, 회고 중 무엇을 먼저 봐야 하는지 다시 추론할 수 있다.
- 유지 기준이 없으면 작은 문서 수정도 여러 문서와 기준 파일을 따로 찾아다니게 되어 마감 비용이 커질 수 있다.

### 범위
- 운영용 조회 응답 반복 검증 스크립트 실패 사례 빠른 참조 적용 예시를 고칠 때 먼저 볼 문서와 같이 고칠 파일 기준을 정리한다.
- 아래 내용을 최소 범위로 포함한다.
  - 유지 기준 정리 전에 적용 예시 문서와 테스트 기준 다시 확인
  - 적용 예시 수정 시 먼저 볼 문서와 같이 갱신할 파일 정리
  - 적용 예시와 상세 예시 표 어긋남 점검 순서 정리
  - 유지 보수 체크리스트와 단계 마감 정리
- 현재 운영용 조회 응답 반복 검증 세트에 적용한다.
  - `manual-rerun-response-seed-command-guide.md`
  - 관련 회고와 단계 요약
  - 스크립트 관련 자동 검증 테스트

### 비대상
- 새 스크립트 구현
- 실제 앱/H2 대표 검증 재실행
- 기존 종료 코드, 출력 파일 이름, 응답 계약 변경
- 운영자용 API 계약 변경
- 관리자 UI
- 빠른 참조 카드 자체의 새 유형 추가

### 외부 계약
- 기존 재실행, 재시도, 단건 조회, 목록 조회, 이력 조회, 관리자 조치 경로와 HTTP 상태 계약은 유지한다.
- 기존 스크립트 이름, 종료 코드, 출력 파일 이름은 이번 단계에서 바꾸지 않는다.
- 이번 단계는 적용 예시 유지 기준 정리 작업이지, 상세 실패 사례 문서나 빠른 참조 카드 자체의 내용을 다시 늘리는 단계가 아니다.

### 핵심 시나리오
1. 유지 기준 정리 전에 적용 예시 문서와 테스트 기준 다시 확인
   - `SPEC-0036` 단계 요약, 현재 적용 예시 문서, 자동 검증 테스트를 다시 읽고 돌려 이번 단계에서 먼저 볼 문서를 정한다.
2. 적용 예시 수정 시 먼저 볼 문서와 같이 갱신할 파일 정리
   - 적용 예시를 고칠 때 빠른 참조, 상세 예시 표, 보조 문서, 회고 중 무엇을 먼저 보고 어떤 파일을 같이 갱신해야 하는지 정리한다.
3. 적용 예시와 상세 예시 표 어긋남 점검 순서 정리
   - 적용 예시, 상세 예시 표, 보조 문서가 서로 어긋날 때 어디서부터 다시 봐야 하는지 순서로 정리한다.
4. 유지 보수 체크리스트와 단계 마감 정리
   - 문서를 고친 뒤 마지막에 무엇을 다시 확인해야 하는지와 단계 마감 기준을 남긴다.

### 작업 분해 기준
- `TASK-0001` 유지 기준 정리 전에 적용 예시 문서와 테스트 기준 확인
- `TASK-0002` 적용 예시 수정 시 먼저 볼 문서와 같이 갱신할 파일 정리
- `TASK-0003` 적용 예시와 상세 예시 표 어긋남 점검 순서 정리
- `TASK-0004` 유지 보수 체크리스트와 단계 마감 정리

### 연결 검증 기준
- `manual-rerun-script-application-maintenance-safety-net-preserved`
- `manual-rerun-script-application-maintenance-update-boundary-documented`
- `manual-rerun-script-application-maintenance-drift-order-documented`
- `manual-rerun-script-application-maintenance-closeout-documented`

### 필수 테스트 시나리오
- 이번 단계 동안 기존 재실행, 재시도, 단건 조회, 목록 조회, 이력 조회, 관리자 조치 자동 검증은 유지된다.
- 운영자가 적용 예시 문서를 고칠 때 무엇을 먼저 보고 무엇을 같이 갱신해야 하는지 문서만 보고 다시 고를 수 있어야 한다.
- 적용 예시와 상세 예시 표, 보조 문서가 어긋날 때 어디서부터 다시 봐야 하는지 문서만 보고 다시 찾을 수 있어야 한다.

## 후속 단계 후보
### ID
SPEC-0038

### 이름
운영용 조회 응답 반복 검증 스크립트 적용 예시 유지 빠른 참조 정리

### 시작 조건
- `현재 활성 단계`가 완료되고, 적용 예시 유지 기준 문서가 정리된 뒤 시작한다.

### 목표
- 유지 기준이 길어졌을 때 운영자가 무엇을 먼저 볼지 더 빨리 좁힐 수 있도록 유지 기준용 빠른 참조를 정리한다.

### 후속 변경 범위
- 적용 예시 유지 기준 빠른 참조
- 유지 체크 카드 또는 비교표
- 유지 기준 상세 문서 이동 예시

### 후속 변경 비대상
- 사용자용 UI
- 장기 저장소 도입

### 후속 검증 방향
- 현재 구현 결과를 바탕으로 운영자가 적용 예시 유지 작업에서 어디부터 시작해야 하는지 더 빨리 판단할 수 있게 한다.
