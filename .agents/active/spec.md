# 현재 활성 단계

## 문서 목적
이 문서는 현재 활성 구현 범위와 후속 구현 범위를 고정하는 현재 단계 문서다.
검증 기준과 작업 문서는 이 문서의 활성 범위를 기준으로 작성하며, GitHub 이슈는 각 작업을 외부에서 추적하기 위한 수단으로만 사용한다.

## 현재 활성 단계
### ID
SPEC-0027

### 이름
운영용 조회 응답 준비 데이터 반복 검증 스크립트 검토

### 목표
- 운영용 조회 응답 대표 검증 절차를 반자동 스크립트로 묶을 가치가 있는지 검토한다.
- 현재 보조 명령 가이드 기준 절차에서 어떤 단계를 스크립트로 묶고, 어떤 단계는 사람이 직접 확인해야 하는지 경계를 정리한다.
- 스크립트 도입 이점, 위험, 비대상을 문서로 고정해 다음 단계 구현 여부를 판단할 근거를 남긴다.

### 대상 문제
- 준비 데이터 가이드와 보조 명령 가이드만으로 대표 검증을 다시 수행할 수는 있지만, 단계가 길고 임시 파일 경로가 많아 반복 비용이 크다.
- 준비 데이터 초기화/적용, 앱 기동, 대표 요청, 실행 키 추출, 앱 종료, H2 조회를 모두 수작업으로 반복하면 누락이나 순서 오류가 생길 수 있다.
- 반대로 이 절차를 전부 자동화하면 실행 흔적 확인과 오류 분리 지점이 흐려질 수 있어, 어떤 단계까지 스크립트로 묶을지 먼저 검토해야 한다.

### 범위
- 운영용 조회 응답 대표 검증 절차를 반자동 스크립트로 묶을 후보 단계를 검토한다.
- 아래 내용을 최소 범위로 포함한다.
  - 현재 대표 검증 절차의 반복 비용과 사람 손이 많이 가는 단계 정리
  - 스크립트 후보 단계와 입력/출력 경계 정리
  - 스크립트가 해도 되는 단계와 사람이 직접 남겨야 하는 근거 단계 분리
  - 스크립트 도입 이점, 위험, 유지 비용, 비대상 정리
  - 다음 단계 구현으로 넘어갈지 판단하는 마감 근거
- 현재 운영용 조회 응답 세트에 적용한다.
  - `rerun`
  - `retry`
  - 단건 조회
  - 목록 조회
  - 이력 조회
  - 관리자 조치 응답

### 비대상
- 실제 스크립트 구현
- 새 경로 추가
- 응답 필드 추가 또는 이름 변경
- 준비 데이터 SQL 자체 구조 변경
- 운영자 UI

### 외부 계약
- 기존 rerun/retry/단건 조회/목록 조회/이력 조회/관리자 조치 경로와 HTTP 상태 계약은 유지한다.
- 기존 응답 필드 의미는 바꾸지 않는다.
- 이번 단계는 대표 검증 스크립트 도입 필요성을 검토하는 작업이며, 관리 API 계약 자체를 바꾸는 작업이 아니다.

### 핵심 시나리오
1. 스크립트 검토 시작 전 기존 근거 확인
   - `SPEC-0026` 대표 검증 회고와 현재 가이드 문서가 스크립트 검토 단계의 시작 근거로 충분한지 먼저 확인한다.
   - 부족한 경우만 최소 문서 보강을 검토한다.
2. 스크립트 후보 단계와 입력/출력 경계 정리
   - 준비 데이터 초기화/적용, 앱 기동, 대표 요청, 파생 실행 키 추출, 앱 종료, H2 조회 중 어떤 단계가 스크립트 후보인지 정리한다.
   - 입력 값, 출력 근거 자료, 사람이 직접 확인해야 하는 지점을 같이 정리한다.
3. 스크립트 도입 이점과 위험 정리
   - 반복 비용 절감, 오류 감소, 실행 근거 일관성 같은 이점을 정리한다.
   - H2 lock, 실제 실행 흔적 가시성 저하, 디버깅 난이도 증가 같은 위험과 비대상을 함께 정리한다.
4. 도입 판단과 다음 단계 마감
   - 현재는 검토로 충분한지, 아니면 후속 spec에서 실제 스크립트 초안을 만들어야 하는지 판단 근거를 남긴다.
   - 대상 테스트와 전체 테스트를 다시 돌려 문서 정리로 기존 검증 흐름이 흔들리지 않았는지 확인한다.

### 작업 분해 기준
- `TASK-0001` 스크립트 검토 시작 전 기존 근거 확인
- `TASK-0002` 스크립트 후보 단계와 입력/출력 경계 정리
- `TASK-0003` 도입 이점과 위험 정리
- `TASK-0004` 도입 판단과 단계 마감

### 연결 검증 기준
- `manual-rerun-script-review-safety-net-preserved`
- `manual-rerun-script-boundary-documented`
- `manual-rerun-script-tradeoff-documented`
- `manual-rerun-script-closeout-completed`

### 필수 테스트 시나리오
- 스크립트 검토 문서를 정리하는 동안 기존 rerun/retry/단건 조회/목록 조회/이력 조회/관리자 조치 자동 검증은 유지된다.
- 현재 보조 명령 가이드와 회고만으로 대표 검증 절차를 다시 재구성할 수 있어야 한다.
- 어떤 단계가 스크립트 후보이고 어떤 단계는 수동 실행 근거 확인으로 남겨야 하는지 문서에서 구분되어야 한다.

## 후속 단계 후보
### ID
SPEC-0028

### 이름
운영용 조회 응답 준비 데이터 반복 검증 스크립트 초안 검토

### 시작 조건
- `현재 활성 단계`가 완료되고, 대표 검증 보조 명령이 문서화된 뒤 시작한다.

### 목표
- `현재 활성 단계`에서 스크립트 도입 가치가 충분하다고 판단되면, 실제 스크립트 초안 범위를 검토한다.

### 후속 변경 범위
- 보조 스크립트 초안 범위 검토
- 입력/출력 파일과 실행 경계 정리
- 문서 기반 절차와 스크립트 기반 절차의 실제 전환 기준 정리

### 후속 변경 비대상
- 사용자용 UI
- 장기 저장소 도입

### 후속 검증 방향
- 현재 검토 결과를 바탕으로 실제 초안이 필요한지 판단하고, 필요하면 그 초안의 최소 범위를 정한다.
