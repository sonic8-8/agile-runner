# 현재 활성 단계

## 문서 목적
이 문서는 현재 활성 구현 범위와 후속 구현 범위를 고정하는 현재 단계 문서다.
검증 기준과 작업 문서는 이 문서의 활성 범위를 기준으로 작성하며, GitHub 이슈는 각 작업을 외부에서 추적하기 위한 수단으로만 사용한다.

## 현재 활성 단계
### ID
SPEC-0026

### 이름
운영용 조회 응답 준비 데이터 적용 보조 명령 정리

### 목표
- 운영용 조회 응답 대표 검증에 반복적으로 쓰는 준비 데이터 적용, 실행 키 추출, H2 조회 보조 명령을 문서로 고정한다.
- 추가 도구 `jq`를 전제하지 않고도 retry 파생 실행 키를 읽고 대표 검증을 반복할 수 있는 기본 명령 예시를 정리한다.
- 준비 데이터 가이드, 실제 SQL 예시 파일, 대표 검증 절차 사이에서 어느 단계에 어떤 명령을 써야 하는지 더 빠르게 따라갈 수 있게 만든다.

### 대상 문제
- 준비 데이터 SQL 자체는 실제로 쓸 수 있는 수준까지 올라왔지만, 반복 검증에 쓰는 명령은 아직 임시 셸 기록과 회고 문서에 흩어져 있다.
- retry 대표 검증은 응답에서 받은 파생 실행 키를 곧바로 다시 써야 하는데, 현재는 환경마다 임시 명령이 달라져 절차가 흔들릴 수 있다.
- H2 조회, 준비 데이터 정리, 앱 기동 전후 점검 순서를 문서만 읽고 반복하기 어렵다.

### 범위
- 운영용 조회 응답 대표 검증에 필요한 보조 명령 예시를 정리한다.
- 아래 내용을 최소 범위로 포함한다.
  - 준비 데이터 적용 또는 준비 데이터 정리 보조 명령 예시
  - rerun 준비 실행과 retry 대표 검증에 공통으로 쓰는 HTTP 호출 보조 명령 예시
  - retry 파생 실행 키 추출 보조 명령 예시
  - H2 셸 또는 동등한 SQL 조회 도구 보조 명령 예시
  - 대표 검증 절차에서 어떤 시점에 어떤 명령을 쓰는지 문서 연결
- 현재 운영용 조회 응답 세트에 적용한다.
  - `rerun`
  - `retry`
  - 단건 조회
  - 목록 조회
  - 이력 조회
  - 관리자 조치 응답

### 비대상
- 새 경로 추가
- 응답 필드 추가 또는 이름 변경
- 준비 데이터 SQL 자체 구조 변경
- 대표 검증 전체 자동화
- 운영자 UI

### 외부 계약
- 기존 rerun/retry/단건 조회/목록 조회/이력 조회/관리자 조치 경로와 HTTP 상태 계약은 유지한다.
- 기존 응답 필드 의미는 바꾸지 않는다.
- 이번 단계는 대표 검증을 더 쉽게 반복하기 위한 보조 명령과 문서를 정리하는 작업이며, 관리 API 계약 자체를 바꾸는 작업이 아니다.

### 핵심 시나리오
1. 보조 명령 시작 전 기존 근거 확인
   - `SPEC-0025` 대표 검증 회고와 현재 가이드 문서가 보조 명령 정리 단계의 시작 근거로 충분한지 먼저 확인한다.
   - 부족한 경우만 최소 문서 보강을 검토한다.
2. 준비 데이터 적용과 정리 보조 명령 정리
   - 대표 검증 전에 준비 데이터 SQL을 적용하거나 정리하는 기본 명령 예시를 문서화한다.
   - 앱 기동 전후 시점이 섞이지 않게 명령 경계를 같이 적는다.
3. 실행 키 추출과 H2 조회 보조 명령 정리
   - retry 응답에서 파생 실행 키를 읽는 기본 명령을 추가 도구 `jq` 없이도 쓸 수 있는 방식으로 정리한다.
   - 대표 검증 뒤 H2 실행 근거를 확인하는 조회 명령 예시를 시나리오별로 정리한다.
4. 보조 명령 기반 대표 검증 반복 검증과 문서 마감
   - 문서화한 보조 명령만으로 rerun/retry 대표 검증 절차를 다시 따라갈 수 있는지 확인한다.
   - 대상 테스트와 전체 테스트를 다시 돌려 문서 정리로 기존 검증 흐름이 흔들리지 않았는지 확인한다.

### 작업 분해 기준
- `TASK-0001` 보조 명령 시작 전 기존 근거 확인
- `TASK-0002` 준비 데이터 적용과 정리 보조 명령 정리
- `TASK-0003` 실행 키 추출과 H2 조회 보조 명령 정리
- `TASK-0004` 보조 명령 기반 대표 검증과 문서 마감

### 연결 검증 기준
- `manual-rerun-seed-helper-safety-net-preserved`
- `manual-rerun-seed-apply-command-documented`
- `manual-rerun-seed-query-command-documented`
- `manual-rerun-seed-helper-representative-verified`

### 필수 테스트 시나리오
- 보조 명령 문서를 정리하는 동안 기존 rerun/retry/단건 조회/목록 조회/이력 조회/관리자 조치 자동 검증은 유지된다.
- 문서화한 보조 명령만으로 준비 데이터 적용, retry 파생 실행 키 추출, H2 실행 근거 확인 순서를 따라갈 수 있다.
- 대표 검증에서 rerun 준비 실행과 retry 파생 실행을 같은 실행 키 기준으로 다시 확인할 수 있다.

## 후속 단계 후보
### ID
SPEC-0027

### 이름
운영용 조회 응답 준비 데이터 반복 검증 스크립트 검토

### 시작 조건
- `현재 활성 단계`가 완료되고, 대표 검증 보조 명령이 문서화된 뒤 시작한다.

### 목표
- 반복적으로 쓰는 대표 검증 절차를 반자동 스크립트로 묶을 필요가 있는지 검토한다.

### 후속 변경 범위
- 대표 검증 보조 스크립트 후보 정리
- 스크립트 도입 이점과 위험 정리
- 문서 기반 절차와 스크립트 기반 절차의 경계 정리

### 후속 변경 비대상
- 사용자용 UI
- 장기 저장소 도입

### 후속 검증 방향
- 현재 문서 기반 절차만으로 충분한지, 아니면 실제로 반복 비용이 높아 스크립트가 필요한지 근거를 수집한다.
