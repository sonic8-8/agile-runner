# 현재 활성 작업 집합

## 문서 목적
이 문서는 현재 활성 범위를 실제 구현 단위로 분해한 작업 문서다.
각 작업은 하나의 명확한 결과, 검증 기준, GitHub 이슈 연결 규칙을 가져야 한다.

## 현재 활성 단계
- ID: `SPEC-0033`
- 이름: `운영용 조회 응답 반복 검증 스크립트 오류 대응 기준 정리`
- 기준 문서:
  - `.agents/active/spec.md`
  - `.agents/criteria/SPEC-0033-manual-rerun-script-error-handling.json`

## 공통 규칙
- 구현 순서는 `TASK-0001 -> TASK-0002 -> TASK-0003 -> TASK-0004`로 고정한다.
- 각 작업 시작 전 해당 작업용 GitHub 이슈를 새로 연결한다.
- 각 작업은 연결된 검증 기준과 테스트 근거가 없으면 완료로 보지 않는다.
- 기존 재실행, 재시도, 단건 조회, 목록 조회, 이력 조회, 관리자 조치 계약은 유지한다.
- 이번 단계는 스크립트 실패 시 운영자가 어디서부터 다시 확인해야 하는지 정리하는 데 집중한다.
- `TASK-0001` 시작 전 `SPEC-0032` 단계 요약 문서와 현재 보조 명령 가이드, 자동 검증 테스트가 오류 대응 기준 정리 시작 안전망으로 충분한지 먼저 확인한다.

## TASK-0001
### 이름
오류 대응 기준 시작 전 기존 근거와 안전망 확인

### 목표
- 반복 검증 스크립트 오류 대응 기준 정리 단계를 시작하기 전에 `SPEC-0032` 마감 근거와 현재 가이드, 자동 검증이 시작 안전망으로 충분한지 확인한다.

### 구현 범위
- 기존 단계 요약, 마지막 회고, 보조 명령 가이드, 관련 테스트를 우선 재사용한다.
- 아래 기준을 먼저 확인한다.
  - 스크립트 오류 대응 기준을 정리할 수 있는 기존 근거 확보
  - 실제 오류 대응 기준 정리 단계로 넘어가도 되는 판단 근거 정리
  - 자동 검증 테스트 유지
  - `SPEC-0032` 단계 요약 문서 연결
- 기존 안전망이 충분하면 근거를 회고에 남기고, 부족한 경우만 최소 문서 보강을 검토한다.

### 비대상
- 새 스크립트 구현
- 실제 앱/H2 대표 검증 재실행

### 연결 검증 기준
- `manual-rerun-script-error-safety-net-preserved`

### 완료 조건
- 스크립트 오류 대응 기준 정리 단계의 시작 근거가 충분하다는 회고와 테스트 근거가 남는다.

### 검증
- 관련 대상 테스트 실행 통과
- 저장소 표준 전체 테스트 명령 통과
- `SPEC-0032` 단계 요약 문서와 대표 검증 회고 경로 명시

### GitHub 이슈
- 새 이슈 생성
- 권장 제목: `[BE] 오류 대응 기준 시작 전 기존 근거와 안전망 확인`
