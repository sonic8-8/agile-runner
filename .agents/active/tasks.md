# 현재 활성 작업 집합

## 문서 목적
이 문서는 현재 활성 범위를 실제 구현 단위로 분해한 작업 문서다.
각 작업은 하나의 명확한 결과, 검증 기준, GitHub 이슈 연결 규칙을 가져야 한다.

## 현재 활성 단계
- ID: `SPEC-0027`
- 이름: `운영용 조회 응답 준비 데이터 반복 검증 스크립트 검토`
- 기준 문서:
  - `.agents/active/spec.md`
  - `.agents/criteria/SPEC-0027-manual-rerun-script-review.json`

## 공통 규칙
- 구현 순서는 `TASK-0001 -> TASK-0002 -> TASK-0003 -> TASK-0004`로 고정한다.
- 각 작업 시작 전 해당 작업용 GitHub 이슈를 새로 연결한다.
- 각 작업은 연결된 검증 기준과 테스트 근거가 없으면 완료로 보지 않는다.
- 기존 `POST /reviews/rerun`, `POST /reviews/rerun/{executionKey}/retry`, `GET /reviews/rerun/{executionKey}`, `GET /reviews/rerun/executions`, `GET /reviews/rerun/{executionKey}/actions/history`, `POST /reviews/rerun/{executionKey}/actions` 계약은 유지한다.
- 이번 단계는 대표 검증 스크립트 도입 필요성과 경계 검토에 집중한다.
- `TASK-0001` 시작 전 `SPEC-0026` 대표 검증 회고와 현재 가이드 문서가 스크립트 검토 단계의 안전망으로 충분한지 먼저 확인한다.

## TASK-0001
### 이름
스크립트 검토 시작 전 기존 근거 확인

### 목표
- 대표 검증 스크립트 검토 단계를 시작하기 전에 `SPEC-0026` 대표 검증 근거와 현재 가이드가 시작 안전망으로 충분한지 확인한다.

### 구현 범위
- 기존 회고와 관련 controller/service 테스트를 우선 재사용한다.
- 아래 기준을 먼저 확인한다.
  - 대표 검증에서 실제로 반복 비용이 컸던 단계 정리
  - 추가 도구 `jq` 의존처럼 환경별로 흔들린 명령 사례 정리
  - 자동 검증 테스트 유지
  - rerun/retry 대표 검증 근거 유지
- 기존 안전망이 충분하면 근거를 회고에 남기고, 부족한 경우만 최소 문서 보강을 검토한다.

### 비대상
- 실제 스크립트 초안 추가
- 실제 앱/H2 대표 검증 재실행

### 연결 검증 기준
- `manual-rerun-script-review-safety-net-preserved`

### 완료 조건
- 스크립트 검토 단계의 시작 근거가 충분하다는 회고와 테스트 근거가 남는다.

### 검증
- 관련 대상 테스트 실행 통과
- 저장소 표준 전체 테스트 명령 통과
- `SPEC-0026 / TASK-0004` 회고 경로 명시

### GitHub 이슈
- 새 이슈 생성
- 권장 제목: `[BE] 스크립트 검토 시작 전 기존 근거 확인`

## TASK-0002
### 이름
스크립트 후보 단계와 입력/출력 경계 정리

### 목표
- 현재 대표 검증 절차에서 스크립트 후보 단계와 입력/출력 경계를 정리한다.

### 구현 범위
- 아래 기준을 문서에 반영한다.
  - 준비 데이터 초기화/적용 중 스크립트 후보 단계
  - 앱 기동, 대표 요청, 파생 실행 키 추출, 앱 종료, H2 조회 중 스크립트 후보 단계
  - 각 단계의 입력 값, 출력 근거 자료, 사람이 직접 봐야 하는 확인 포인트
  - 문서 기반 절차와 스크립트 기반 절차를 섞지 않기 위한 경계

### 관련 파일 후보

- 위 회고 문서는 `SPEC-0026`에서 대표 검증을 실제로 다시 수행한 근거 문서다.

### 비대상
- 실제 스크립트 구현
- 실제 앱/H2 대표 검증 재실행
- 새 보조 명령 추가

### 연결 검증 기준
- `manual-rerun-script-boundary-documented`

- 문서만 읽고 입력/출력 경계를 재구성했을 때 빠진 단계가 없는지 확인한 리뷰 근거가 남는다.

### 검증

- 필요하면 검토 문서를 최소 보정한다.
